Piano Practice Algorithm (28 bars)

Deterministic daily workflow for 2-bar chunking with backward chaining, spaced recall, and integration. Optimize for retention and reliable transitions, not comfort.

Chunk size: 2 bars
Total chunks: 14
Direction: from end → start
Focus: recall + binding

Core principles (non-negotiable)

1
New material = massed practice. Today’s chunk gets the repetitions.
2
Old material = spaced recall. Low-volume, high-signal retrieval prevents decay.
3
Integration is a separate task. Chunk mastery ≠ piece mastery.
4
Recall failure triggers repair, not brute force. Fix the defect; stop once recall works.

Daily practice template (baseline)

Phase A — Recall (old material)

2–3 reps / chunk
  • Play each previously learned chunk 2–3 times.
  • No warm-up for these; treat them as retrieval tests.
  • If clean → move on immediately.
  • If unstable → mark for Phase C.

Phase B — Acquisition (new chunk)

20–30 reps
  • Today’s new 2 bars get 20–30 focused repetitions.
  • Slow → accurate → stable (hands separate if needed).
  • Stop when attention drops; don’t grind past that point.

Phase C — Repair (only if needed)

5–10 reps
  • Only for chunks that failed during Phase A.
  • Do 5–10 targeted repetitions addressing the exact failure mode.
  • Exit as soon as recall succeeds once.

Weekly macro-structure

This prevents “isolated chunks.”

Every 2 days: adjacency binding

After Phase B, practice transitions:

  • Chunk 1 → 2, 2 → 3, … up to the newest chunk
  • 2–3 reps per pair
  • No stopping between chunks

Every 4 days: chain expansion

Replace adjacency binding with chained endings:

  • Last 4 bars
  • Last 8 bars
  • Last 12 bars (as available)
